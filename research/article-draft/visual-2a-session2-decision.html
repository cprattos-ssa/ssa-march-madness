Session 2: Post-Mortem Analysis
Player data experiment + injury analysis
TRIGGER
Tournament starts. Model misses key upsets on Day 1.
3 games predicted at 90%+ confidence for the wrong team, accounting for 64% of Day 1 prediction error
Could individual player stats have predicted these upsets?
PARALLEL
Pull player data
Barttorvik API: 30K records spanning 6 seasons, 99.3% match rate
PARALLEL
Audit existing features
67 files checked: confirmed no player-level data in pipeline
PARALLEL
Pull live scores
ESPN results vs. model predictions for real-time validation
EXPERIMENT
Engineer 8 player features, retrain 30 configurations across 3 architectures
Team WAR, Star WAR, senior %, avg class year, roster height, star dependency, experience mix
RESULT
Brier score: 0.0503 → 0.0544. Every configuration got worse.
Player features correlated below r = 0.12 with outcomes. Not one cracked the top 15 in feature importance.
INSIGHT
The irony: features that hurt the model are exactly what flagged the upsets.
The aggregate model doesn't need player data - adding it made every configuration worse. But focused player data (injuries, missing starters) is exactly what the edge cases need.
VALIDATED
Research & validation
Kaggle winners confirmed: no player data used. 100-game minimum for reliable stats. Probability clipping standard practice.
VALIDATED
Injury analysis
8 players out pre-tipoff. Duke, Gonzaga, BYU predictions manually adjusted. Injury-adjusted numbers closer to truth.
